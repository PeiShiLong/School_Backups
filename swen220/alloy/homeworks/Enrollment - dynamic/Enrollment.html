Overview

Consider a course enrollment system, where each Course has a set of Persons enrolled, a limit on the number of Persons enrolled, and a set of prerequisite Courses that must be completed before taking the given course. Each Person has a set of completed Courses on a transcript. The signatures, relations, and facts defining the rules of the system are in the file Enrollment-static.als.

Problem

Copy the static model to file Enrollment-dynamic.als and modify it to create a dynamic model showing legitimate enrollments and withdrawals.

  • Add the ordered Step set.
  • The only relation that can change from one Step to the next is enrolled; thus the only facts that must be converted to step-varying predicates are those that refer to enrolled.
  • Convert only the necessary facts to predicates based on the Step argument.
  • Create an Invariant[st : Step] predicate as the conjunction of the converted predicates.
  • Create an init[st : Step] predicate where the initial state is one with no Person enrolled in any Courses.
    • Define the closure assertion and check for up to size 6 with exactly 1 Step.
    • Define the existence predicate and run it using the same sizes as for the assertion.
  • Create an enroll[p : Person, c : Course, st : Step] predicate to enroll Person p in Course c at Step st. Person p must not already be enrolled in c at Step st.
    The preconditions must be strong enough to ensure the state after enrollment satisfies the Invariant. If the preconditions are all true, then p should be enrolled in c at the next Step and nothing else should change.
    • Define the closure assertion and check for up to size 6 with exactly 2 Step.
    • Define the existence predicate and run it using the same sizes as for the assertion.

Deliverables

Submit only one file, your dynamic model named exactly Enrollment-dynamic.als, to the Enrollment drop box by the specified due date.
